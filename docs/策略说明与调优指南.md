## 1. 当前问题诊断

以 601766（中国中车）回测结果为例，收益极低（最好的趋势策略仅 +0.90%），经分析主要有 **6 个瓶颈**：

### 1.1 预热期过长，浪费有效交易日

| 参数 | 当前值 | 影响 |
|------|--------|------|
| `warmup_period` | 512 天 | 总共 969 个交易日，512 天仅用于"预热"，**实际只交易了 457 天** |

从权益曲线图中可以看到，策略在前 150 个交易日（即原始数据第 512~662 天）捕获了一波上涨至 1.13M，之后持续回撤。这意味着整段行情的前半段（股价从 ~4 元涨到 ~6 元）完全被跳过了。

**建议：** 降至 200，技术指标（最长的 MA60）只需 60 天即可计算，200 天足够。

### 1.2 仓位太保守

| 参数 | 当前值 | 影响 |
|------|--------|------|
| `position_size` | 0.3 (30%) | 即使信号正确，单笔也只动用 30% 资金，70% 闲置 |

从交易分布图看，个别交易盈利 12%+，但由于仓位只有 30%，对总权益的贡献仅为 12% × 30% = 3.6%。

### 1.3 止损止盈过紧

| 参数 | 当前值 | 影响 |
|------|--------|------|
| `stop_loss` | 0.05 (5%) | A 股日涨跌幅限制 10%，5% 止损频繁触发 |
| `take_profit` | 0.10 (10%) | 刚好一个涨停板就止盈，错过后续趋势 |

从回撤图可以看到，策略在后半段持续处于回撤状态（8~12%），说明反复止损造成了净值的持续磨损。

### 1.4 重平衡频率不合理

| 参数 | 当前值 | 影响 |
|------|--------|------|
| `rebalance_freq` | 5 天 | 每 5 个交易日才重新预测一次，对快速行情反应迟钝 |

中国中车作为周期股，日内波动大。5 天的重平衡间隔意味着一周只决策一次，错过了很多入场和离场窗口。

### 1.5 模拟预测器过度外推

从 Prediction vs Actual 图可以清晰看到：预测线（虚线）几乎总是向当前趋势方向过度外推。当价格 6.6 元时预测冲到 7.5+，实际却回落。这是模拟预测器 `trend_weight=0.6` 过高导致的。

### 1.6 未适配 A 股交易规则

当前系统允许做空（SIGNAL_SELL 会开空仓），但 A 股散户不能融券做空。做空信号应该只用于平多仓，不应开新空仓。

## 2. 系统架构与数据流

```
config.py（全局参数）
    │
    ▼
data_loader.py ─── akshare/yfinance 下载行情
    │                   │
    │              compute_features() 计算 15 个技术指标
    ▼                   │
macro_analyzer.py      │
    │  VIX + 上证综指   │
    │  → macro_score    │
    ▼                   ▼
model_wrapper.py    features dict
    │  SimulatedPredictor / TimesFMPredictor
    │  → point_forecast (点预测)
    │  → quantile_forecast (分位数预测)
    ▼
strategy.py ← 读取 features + macro_score + forecast
    │  → SIGNAL_BUY / SIGNAL_SELL / SIGNAL_HOLD
    ▼
backtester.py
    │  滚动窗口回测：开仓 → 止损/止盈 → 平仓
    │  → equity_curve, trades, metrics
    ▼
visualizer.py → 6 张图表输出到 output/
```

### 关键数据流

1. **技术指标（features）**：MA5/20/60、EMA12/26、RSI14、MACD、布林带、ATR 共 15 个
2. **宏观评分（macro_score）**：[-1, +1] 区间，>0 乐观，<0 悲观
3. **模型预测**：
   - `point_forecast`：未来 N 天价格点预测（shape: `[horizon]`）
   - `quantile_forecast`：分位数预测（shape: `[horizon, 10]`），10 个通道分别为 mean, 10%, 20%, ..., 90%

## 3. 三大策略机制详解

### 3.1 趋势预测策略（TrendStrategy）

**核心逻辑：** 如果模型预测价格未来会涨超过阈值，就买入；预测会跌超过阈值，就卖出。

```
计算方式：
  forecast_mean = mean(point_forecast)        # 预测均价
  expected_return = (forecast_mean - 当前价) / 当前价
  end_return = (forecast[-1] - 当前价) / 当前价
  combined_return = 0.6 × expected_return + 0.4 × end_return

宏观调节：
  buy_threshold  = threshold × (1 - macro × macro_influence)
  sell_threshold = threshold × (1 + macro × macro_influence)

信号条件：
  买入: combined_return > buy_threshold  AND  RSI < 75
  卖出: combined_return < -sell_threshold AND  RSI > 25
```

**影响该策略的参数：**

| 参数 | 位置 | 作用 | 调优方向 |
|------|------|------|----------|
| `trend_threshold` | STRATEGY_CONFIG | 买卖信号触发门槛 | ↓ 降低 = 更敏感，交易更频繁 |
| `macro_influence` | STRATEGY_CONFIG | 宏观因子调节幅度 | ↑ 增大 = 宏观因子权重更高 |
| `horizon` | MODEL_CONFIG | 预测步长 | 短 horizon → 更短线 |
| `trend_weight` | SimulatedPredictor | 模拟器的趋势跟随强度 | ↓ 降低 = 减少过度外推 |

### 3.2 分位数风控策略（QuantileRiskStrategy）

**核心逻辑：** 不仅看预测方向，还看预测的不确定性。只有当下行风险可控（低 10% 分位不超阈值）且风险收益比 > 2:1 时才交易。

```
计算方式：
  取 quantile_forecast 末端的分位数行：
    median  = q_end[5]    （50% 分位）
    low_10  = q_end[1]    （10% 分位，最悲观情况）
    high_90 = q_end[9]    （90% 分位，最乐观情况）

  upside   = (high_90 - 当前价) / 当前价
  downside = (当前价 - low_10) / 当前价
  risk_reward = upside / downside

信号条件（做多）：
  downside < adj_risk_threshold    # 下行风险可控
  AND risk_reward > 2.0            # 风险收益比 > 2:1
  AND median_return > 0.005        # 中位数预测看涨
  AND uncertainty < 0.15           # 预测区间不能太宽
```

**影响该策略的参数：**

| 参数 | 位置 | 作用 | 调优方向 |
|------|------|------|----------|
| `quantile_risk_threshold` | STRATEGY_CONFIG | 可接受的最大下行风险 | ↑ 放宽 = 容忍更大亏损但信号更多 |
| `noise_scale` | SimulatedPredictor | 分位数区间宽度 | ↓ 缩小 = 预测区间更窄，更容易过关 |
| `uncertainty < 0.15` | 硬编码 | 最大不确定性阈值 | ↑ 放宽 = 允许更宽的预测区间 |
| `risk_reward > 2.0` | 硬编码 | 最低风险收益比 | ↓ 降低 = 更容易触发买入 |

### 3.3 多 Horizon 集成策略（EnsembleStrategy）

**核心逻辑：** 用 3 个不同的预测窗口（3/5/10 天）分别投票，再加上技术指标确认和宏观投票，多数一致时才交易。

```
计算方式：
  对 horizon = [3, 5, 10]，分别计算 sub_return:
    sub_return = (mean(forecast[:h]) - 当前价) / 当前价
    > 0.5%  → 买入票 (权重 0.2 / 0.3 / 0.5)
    < -0.5% → 卖出票
    其他    → 持有票

  技术确认加分 tech_boost:
    MACD_hist > 0  → +0.1
    价格 < 布林下轨 → +0.1
    宏观因子       → ±(macro × 0.15)

  最终:
    buy_ratio  = 买入权重占比 + tech_boost
    sell_ratio = 卖出权重占比 - tech_boost

信号条件：
  买入: buy_ratio > 0.55
  卖出: sell_ratio > 0.55
```

**影响该策略的参数：**

| 参数 | 位置 | 作用 | 调优方向 |
|------|------|------|----------|
| `ensemble_horizons` | STRATEGY_CONFIG | 多个预测窗口 | 短窗口组合更灵活 |
| `ensemble_weights` | STRATEGY_CONFIG | 各窗口权重 | 加大短窗口权重 = 更灵敏 |
| `> 0.55` 阈值 | 硬编码 | 一致性要求 | ↓ 降低 = 更容易达成一致 |
| `0.005` 投票门槛 | 硬编码 | 单窗口买卖最小收益率 | ↓ 降低 = 更容易产生非 HOLD 票 |

## 4. 参数调优完整手册

以下是 `config.py` 中所有可调参数及其影响：

## 5. A 股市场适配要点

### 5.1 交易规则差异

| 规则 | A 股 | 美股 | 系统现状 |
|------|------|------|----------|
| 日涨跌幅 | ±10% (主板) | 无限制 | 未考虑 |
| T+1 | 当天买入次日才能卖出 | T+0 | 未实现 |
| 最小交易单位 | 100 股（1 手） | 1 股 | 未实现 |
| 做空 | 一般不可（需融券） | 可以 | **允许做空（需修复）** |
| 印花税 | 卖出 0.05% | 无 | 未区分 |
| 交易时间 | 9:30-15:00 | 9:30-16:00 | 不影响日线回测 |

### 5.2 重要的代码级修复建议

**禁止散户做空：** 当前 `backtester.py` 中做空信号会开空仓（`direction = -1`），在 A 股中应该只用于平掉多头仓位，不应新开空仓。修改 `backtester.py` 中的执行逻辑：

```python
# 对 A 股仅允许做多
if data.get("market") == "CN":
    if signal == SIGNAL_SELL and position is None:
        pass  # A 股不允许裸空
    elif signal == SIGNAL_SELL and position is not None and position.direction == 1:
        # 平掉多头仓位
        cash, position = self._close_position(...)
```

**100 股整手交易：**

```python
# A 股需要按 100 股取整
if data.get("market") == "CN":
    shares = (shares // 100) * 100
```

## 7. 进阶策略思路

### 7.1 趋势跟踪 + 动态止盈（Trailing Stop）

当前止盈是固定百分比（达到就平仓），这会在大趋势行情中过早离场。

**改进方案：** 引入移动止盈，随价格上涨不断抬高止损线：

```
持仓后最高价 = max(所有持仓期价格)
动态止损线 = 最高价 × (1 - trailing_pct)

当前价 < 动态止损线 → 平仓

trailing_pct 建议值: 0.05~0.08
```

效果：在 601766 的大涨行情中（4 元→8 元），策略不会在 +10% 就离场，而是一路持有到趋势反转。

### 7.2 波动率自适应仓位（Volatility Targeting）

不同时期波动率差异很大，固定仓位不合理。

```
target_vol = 0.15   # 目标年化波动率 15%
recent_vol = std(最近20天日收益率) × sqrt(252)
position_size = target_vol / recent_vol

低波动时期 → 加仓（例如 0.8）
高波动时期 → 减仓（例如 0.3）
```

### 7.3 突破策略（Breakout Strategy）

利用布林带和 ATR 做突破交易，适合 A 股的涨停板效应：

```
信号条件：
  买入: 收盘价 > 布林上轨 AND 成交量 > 20日均量×1.5 AND ATR 扩大
  卖出: 收盘价 < 布林下轨 OR ATR 收缩

原理: A 股缩量横盘后放量突破，往往是主力启动信号
```

### 7.4 均值回归策略（Mean Reversion）

适合震荡市（A 股大部分时间处于震荡）：

```
信号条件：
  买入: RSI < 30 AND 价格 < MA60 × 0.95 AND MACD金叉
  卖出: RSI > 70 AND 价格 > MA60 × 1.05

配合分位数预测：
  只在 quantile 下行风险可控时才入场
```

### 7.5 多因子轮动策略

在多只股票间选择最强势的持有：

```
每个 rebalance 周期计算每只股票的复合得分：
  score = 0.4 × momentum_20d + 0.3 × (1 - volatility) + 0.3 × model_forecast

持有得分最高的 1~2 只股票，定期轮换
```

适合的股票池示例：`["600519", "000858", "601318", "000001", "601766"]`

### 7.6 信号过滤器增强

在现有策略基础上增加过滤条件，过滤掉低质量信号：

```
成交量过滤: 只在成交量 > MA20(volume) 时交易
趋势过滤:   只在 MA5 > MA20 时做多（顺势交易）
波动过滤:   ATR > 历史均值时不交易（避开剧烈波动期）
时间过滤:   避开财报季前后 5 天（信息不对称严重）
```
